    Network Computing

    Exercises - Intro and Application layer

    1. Cite an advantage and a disadvantage of a tree topology.

    1. Advantage: 1. Easy to detect errors. 2. Sturdiness, if one node is defected other nodes won’t be affected. 3 Easy expansions. 4. Device support, easy to add new device into the tree system and access those devices. 5. Low cable requirement.

    2. Disadvantage: 1. The process of installing the tree topology is difficult, expensive, and hard to handle. 2. The security of tree topology is weak. Easy to hack. 3. When expand a tree, the cost is expensive. 4. Hard to maintenance.

    2. Why is the Internet considered a “network of networks”?

    1. The internet is called a network of network because it’s a global network of computers that are linked together by cables and telephone lines. It makes communication possible among them. It can be defined as global network over a million smaller heterogeneous computer networks. And each smaller network connects to each other. And internet is a set of lots of smaller networks.

    3. Compare and contrast a home and an enterprise network.

    1. Home: 1. All devices and PC’s are equal members of a network. No device is capable of network management. 2. Hosts are often connected to a router. The router connects to a cable or DSL modem.

      1. Enterprise:  Mix of wired, wireless link technologies, connecting a mix of switches and routers. It has ethernet switch. Enterprise network links to ISP, institutional mail, and Web servers. Enterprise network has at least one server.

    4. Consider the network below. What is the total end-to-end delay of a Web request from H1 to H2.

    H1 ————– S ————– H2

    1. Total delay = Queueing delay + nodal processing delay + transmission delay + propagation delay.

    5. Cite two motivations for a content provider (e.g., Facebook) to create their own network to interconnect their data centers.

    1. Improve efficiency, if users directly connect to their data center it will cut the time of sharing bandwidth with other companies and it’s quicker.
    2. Cut costs: Facebook is a big company, if it rents a lot of servers to serve their customers. The price will be huge. They have their own data center as server at least will decease some of the costs.

    6. Can a router route a segment? Why?

    1. No. Segment is the message at transport layer. But a router only implements network layer and below. It only can route a datagram and lower.

    7. What is the main factor limiting the distance of a wired link?

    1. The limit on material and the cost. 2. The disturbance during the long-distance communications. 3. Hard to maintenance.

    8. What is jitter? How could you reduce it?

    1. Jitter: Jitter is when there is a time delay in the sending data packets over network connection. This is often caused by network congestion, and sometimes route changes.

      1. We can use client-side buffer to store data ahead. And use the data in the buffer to maintain continuous transmission during the streaming.

    9. For a communication session between a pair of processes, which process is the client and which process is the server?

    1. Client Process initiates communication.

      1. Server process: Process that waited to be connected.

    10. Cite one advantage and one disadvantage of a P2P application compared to a client-server one.

    1. Advantage: Easy and simple to set up only requiring a hub or a switch to connect all computers together.

      1. Disadvantage:  Security is not good other than setting passwords for files that you don't want people to access. If the connections are not connected to the computers properly then there can be problems accessing certain files. It does not run efficient if you have many computers, it is best to used two to eight computers.

      1. processing cryptocurrency transactions: TCP, when relate to money we want a reliable data transfer communication. And only TCP can have reliable data transfer ability.

    2. checking the weather forecast
      1. UDP. Since the result is not that important. We want to see the results as soon as possible.

    1. Server maintain no information about past client. It will forget any connection after the send the object and connection is closed.

    1. Transmission order of requested objects based on client specified object priority.

    2. Divide objects into binary frames, and schedule frames to mitigate HOL blocking.

    3. Sometimes we want important objects delivered first.

    4. To prevent small objects keep waiting for a big object is transmitting.

    7. TCP waits for up to three duplicate ACKs before executing a fast retransmission. What is the rationale behind this number? For example, why did the TCP designers not choose to retransmit the packet after the first duplicate ACK?

    1. Since TCP does not know whether a duplicate ACK is caused by a lost segment or just a reordering of segments. If three or more duplicate ACKs are received in a row, it is a strong indication that a segment has been lost.

    12. Compare network-assisted and end-to-end based congestion control approaches. In particular, state advantages and disadvantages and an example for each of them.

    1. AD: The router who gives the congestion feedback has a better view of the real congestion that is happening currently. It will give better result on solving congestions.

    End-to-End:

    1. DIS: The feedback may not be accurate, since we are relying on duplicate ACKs or packet loss.
    2. AD: this method will work in most scenarios. And resist to router’s failure.

    5. Compare bus based and interconnection network based switches. Cite at least one advantage and one disadvantage for each of them.

    1. Bus:
      1. Ad: An input port transfers a packet directly to the output port over a shared bus without intervention by the routing processor. And one packet can use all bandwidth by itself.
      2. Dis: It only can send one packet at a time. Other packets must wait at input ports.
    2. Interconnection:
      1. Ad: It can send multiple packets during the same time.
      2. Dis: It still can be block by when two packets’ forwarding routes are crossed. But the bandwidth is lower since it needs to share the bandwidth with other routes.

    1. Two dropping packets policies:
      1. Drop the arriving packet as drop-tail

    2. I would chose the policy that will drop tail packets. Since, if we mainly use web browsing, we don’t want to miss anything from our web page. It will confused us by the missing contents. And when we watch video instead of video streaming. We don’t miss anything either.

    2. additionally, real-time voice-over-IP packets might receive priority over non-real-time traffic such e-mail packets
    1. For example, they may want to set high-priority on packets that are carrying network management information.
    2. And set high-priority on real-time voice-over-IP packets than non-real time traffic such as e-mails.

    8. What is the difference between Round Robin (RR) and Weighted Fair Queueing (WFQ) packet scheduling policies? Is there a case where RR and WFQ will behave exactly the same?

    1. Difference:
      1. In round robin, each class has same chance to get out of the queue by cycling each class.
      2. But in weighted Fair queueing, each class has been assigned with a weight. And each class may receive a differential amount of service in any interval of time based on its weight.
    2. Yes: when every class has the same weight.

    10. When a large datagram is fragmented into multiple smaller datagrams at the network layer, where are these smaller datagrams reassembled into a single larger datagram?

    B or D

    (a) the next router

    (b) the last router in the path to the destination

    (c) the receiver

    (d) the packet is reassembled only at the transport layer on the receiver side

    These three fields have to do with so-called
    IP fragmentation, when a large IP datagram is broken into several smaller IP datagrams

    12. What is meant by the term “route aggregation”? Why is it useful for a router to perform route aggregation?

    1. This ability to use a single prefix to advertise multiple networks is often referred to as address aggregation (also route aggregation or route summarization).
    2. Route aggregation lets you take several specific routes and combine them into one inclusive route. Route aggregation can reduce the number of routes a given protocol advertises.
    1. It means that one main network can have multiple sub network inside of it. And the rest of world can access those sub networks by the main network IP prefix without knowing it has multiple sub networks.
    2. It allows efficient advertisement of routing information.

    13. Cite four differences between IPv4 and IPv6. For two of these differences, explain what was the main motivation behind them

    1. Difference:

      1. IPv4 is 32 bit IP address whereas IPv6 is 128 Bit address
      2. IPv4 has options like upper-layer, IPV6 has no options

    19. Compare and contrast link-state and distance-vector routing algorithms.

    1. Link state algorithms are global algorithms. All routers have complete topology, link cost info. And in each calculation of best routes, it will consider all nodes.
    2. Distance-vector routing algorithms are decentralized algorithms. Iterative process of computation, exchange of info only with neighbors.
    1. Bellman-Ford algorithm is used for performing distance vector routing whereas Dijsktra is used for performing the link state routing.
    2. In distance vector routing the routers receive the topological information from the neighbour point of view. On the contrary, in link state routing the router receive complete information on the network topology.
    3. Distance vector routing calculates the best route based on the distance (fewest number of hops). As against, Link state routing calculates the best route on the basis of least cost.
      Link state routing updates only the link state while Distance vector routing updates full routing table.
    4. The frequency of update in both routing technique is different distance vector update periodically whereas link state update frequency employs triggered updates.

    20. What is the count-to-infinity problem in distance vector routing? How does it affect routing on the Internet?

    1. If node A tells node B that it has a path somewhere, there is no way for node B to know if the path has node B as a part of it.
    1. CTI is a routing loop problem happens when the cost of one link in the network increased, and the affected node tell its neighbors to update the distance vector. But its neighbor’s route has the affected node’s old route’s distance. Hence, its neighbor will update its distance again and again until the cost is larger or equal than the current cost of the affected node. If the cost of that link increased to infinity or very large, the loop will continue for a very long time.
    2. It will cost a lot of resources and time for routers updating their link cost for nothing. And this will make the packet transmission slower if the path hasn’t been updated to the right cost. The packets will follow the old path with a high cost to destination.

    8. What are RTS and CTS frames? Why are they used in a WiFi network?

    1. RTS: request to send Frame, it is a control frame to reserve access to the channel with a period of time.
    2. CTS: clear to send Frame, it gives the sender explicit permission to send and also instructs the other stations not to send for the reserved duration.
    3. It is aimed for solving the collision that caused by hidden terminal problem.

    In order to avoid this problem, the IEEE 802.11 protocol allows a station to
    use a short Request to Send (RTS) control frame and a short Clear to Send (CTS)
    control frame to reserve access to the channel. When a sender wants to send a DATA
    frame, it can first send an RTS frame to the AP, indicating the total time required
    to transmit the DATA frame and the acknowledgment (ACK) frame. When the AP
    receives the RTS frame, it responds by broadcasting a CTS frame. This CTS frame serves two purposes: It gives the sender explicit permission to send and also instructs
    the other stations not to send for the reserved duration.

    12. Cite one advantage and one disadvantage of channel partitioning, random access and “taking turns” medium access protocols.

    Channel partitioning:

    TDM is appealing because it eliminates collisions and is perfectly fair: Each
    node gets a dedicated transmission rate of R/N bps during each frame time. However,
    it has two major drawbacks. First, a node is limited to an average rate of R/N bps
    even when it is the only node with packets to send. A second drawback is that a node
    must always wait for its turn in the transmission sequence—again, even when it is
    the only node with a frame to send.

    1. Channel Partitioning:
      1. Advantage: It eliminates collisions and is perfectly fair.
      2. Disadvantage:
        1. A node is limited to an average rate of R/N bps, even when it is the only node with packets to send.
        2. A node must always wait for its turn in the transmission sequence. even there is only one node in the sequence.

    Random Access protocols:

    The second broad class of multiple access protocols are random access protocols.
    In a random access protocol, a transmitting node always transmits at the full rate
    of the channel, namely, R bps. When there is a collision, each node involved in the
    collision repeatedly retransmits its frame (that is, packet) until its frame gets through
    without a collision. But when a node experiences a collision, it doesn’t necessarily
    retransmit the frame right away. Instead it waits a random delay before retransmitting
    the frame. Each node involved in a collision chooses independent random
    delays. Because the random delays are independently chosen, it is possible that one
    of the nodes will pick a delay that is sufficiently less than the delays of the other colliding
    nodes and will therefore be able to sneak its frame into the channel without a
    collision.

    1. AD: Every node transmission is at maximum/full rate of the channel.
    2. Disadvantage: If in the worst scenario during a collision, more than one nodes wait a same random delay time to retransmit. This condition will cause another collision, and another collision possibly after this collision if this scenario happens again.

    Taking-Turns Protocols:

    The first one is the polling protocol. The polling protocol requires one of the
    nodes to be designated as a master node. The master node polls each of the nodes in
    a round-robin fashion. In particular, the master node first sends a message to node 1,
    saying that it (node 1) can transmit up to some maximum number of frames. After node
    1 transmits some frames, the master node tells node 2 it (node 2) can transmit up to the
    maximum number of frames. (The master node can determine when a node has finished
    sending its frames by observing the lack of a signal on the channel.) The procedure continues
    in this manner, with the master node polling each of the nodes in a cyclic manner.

    The polling protocol eliminates the collisions and empty slots that plague random
    access protocols. This allows polling to achieve a much higher efficiency. But it also has a few drawbacks. The first drawback is that the protocol introduces a
    polling delay—the amount of time required to notify a node that it can transmit. If,
    for example, only one node is active, then the node will transmit at a rate less than
    R bps, as the master node must poll each of the inactive nodes in turn each time the
    active node has sent its maximum number of frames. The second drawback, which is
    potentially more serious, is that if the master node fails, the entire channel becomes
    inoperative. The Bluetooth protocol, which we will study in Section!6.3, is an example
    of a polling protocol.

    The second taking-turns protocol is the token-passing protocol. In this protocol
    there is no master node. A small, special-purpose frame known as a token is
    exchanged among the nodes in some fixed order. For example, node 1 might always
    send the token to node 2, node 2 might always send the token to node 3, and node
    N might always send the token to node 1. When a node receives a token, it holds
    onto the token only if it has some frames to transmit; otherwise, it immediately forwards
    the token to the next node. If a node does have frames to transmit when it
    receives the token, it sends up to a maximum number of frames and then forwards
    the token to the next node. Token passing is decentralized and highly efficient. But
    it has its problems as well. For example, the failure of one node can crash the entire
    channel. Or if a node accidentally neglects to release the token, then some recovery
    procedure must be invoked to get the token back in circulation. Over the years many
    token-passing protocols have been developed, including the fiber distributed data
    interface (FDDI) protocol [Jain 1994] and the IEEE 802.5 token ring protocol [IEEE
    802.5 2012], and each one had to address these as well as other sticky issues.

    1. AD: It avoids collision and node uses full transmission rate.
    2. DIS: Nodes with more to send can take longer turns.
